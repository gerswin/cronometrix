<objective>
Fix the Phase 7 test compile regression: `cargo test --no-run` currently fails on 14 test crates with `error[E0063]: missing fields backfill_tx, captures and purge_tx in initializer of AppState`. Phase 7 (07-01) added 3 new fields to `AppState` (`purge_tx`, `backfill_tx`, `captures`) but did not update legacy test files that build `AppState { ... }` literals.

Purpose: Restore green test compilation AND prevent the same regression from recurring whenever `AppState` grows again. The fix is twofold — patch every broken site, AND extract a single shared helper so future field additions only need to be wired into one place.

Output:
- A new `pub fn test_state(db, config) -> AppState` in the existing `backend/tests/common/mod.rs`, populating all optional fields with `None` and `captures` with a fresh `new_captures_map()`.
- 14 test files refactored to call `test_state(...)` instead of inlining `AppState { ... }`.
- `cargo test --no-run` exits 0; `cargo nextest run` (or `cargo test`) passes with no regressions.
</objective>
