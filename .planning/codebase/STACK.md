# Technology Stack

**Analysis Date:** 2026-04-19

## Languages

**Primary:**
- Rust 2021 edition - Backend API server and core business logic
- TypeScript 5.x - Frontend application and type safety
- SQL - Database schema and migrations

**Secondary:**
- JavaScript/JSX - React component runtime
- Bash - Deployment scripts and tooling

## Runtime

**Environment:**
- Rust async runtime via Tokio 1.51.x - High-performance concurrent request handling
- Node.js (via Next.js) - Frontend development and build

**Package Manager:**
- Cargo 1.77+ - Rust dependency management
- npm - Node.js package management (see `frontend/package.json`)
- Lockfile: Present (`Cargo.lock`, `frontend/package-lock.json` or equivalent)

## Frameworks

**Core:**
- Axum 0.8.8 - HTTP server framework, routing, extractors; Tokio-native with Tower middleware
- Next.js 16.2.3 - React framework with App Router, SSR/SSG for admin dashboard
- React 19.2.4 - UI library with concurrent features and Server Components support

**Testing:**
- axum-test 16 - Testing utilities for Axum handlers (backend)
- Vitest (not yet in package.json but recommended) - Frontend unit/integration tests with Vite

**Build/Dev:**
- Tokio 1.51.x - Async runtime for backend
- Tower 0.5 - HTTP middleware abstraction layer
- Tower-HTTP 0.6.x - Production middleware (CORS, tracing, compression, timeout)

## Key Dependencies

**Critical:**

Backend:
- libsql 0.9.30 - SQLite local + Turso cloud sync via embedded replica
- jsonwebtoken 10.3.0 - JWT creation/validation (HS256) for authentication
- axum-extra 0.12.5 - Cookie handling for refresh token storage
- password-auth 1.0 - Argon2id password hashing (timing-safe)
- serde 1.0 + serde_json 1.0 - Request/response serialization
- chrono 0.4.42 - Date/time arithmetic for attendance calculations
- validator 0.20 - Request payload validation with derive macros
- tracing 0.1 + tracing-subscriber 0.3 - Structured logging with JSON output
- thiserror 2.x - Typed error types that implement IntoResponse
- anyhow 1.x - Error propagation in application code
- uuid 1.x - UUID v4 ID generation

Frontend:
- @tanstack/react-query 5.99.0 - Server state management, caching, background refetch
- @tanstack/react-table (not yet in package.json) - Headless data tables (planned for employee/timesheet lists)
- react-hook-form 7.72.1 - Form state management (uncontrolled, zero re-renders)
- zod 4.3.6 - Schema validation + TypeScript type inference
- axios 1.15.0 - HTTP client to Rust backend
- shadcn/ui (via shadcn 4.2.0) - Component library (forms, dialogs, cards) built on Radix UI
- tailwindcss 4.x - Utility CSS framework (required by shadcn)
- lucide-react 1.8.0 - Icon set for UI components
- class-variance-authority 0.7.1 - Variant management for components
- clsx 2.1.1 - Conditional className utility

**Infrastructure:**
- dotenvy 0.15 - Environment variable loading (maintained fork of dotenv)
- dotenv (via dotenvy) - Configuration from .env files (see `/backend/.env.example`)

## Configuration

**Environment:**
- Backend: `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`, `JWT_SECRET` (required, min 32 chars), `SERVER_HOST`, `SERVER_PORT`, `TURSO_SYNC_INTERVAL`
- Frontend: `NEXT_PUBLIC_API_URL` (defaults to `http://localhost:3001`)
- See `backend/.env.example` for all required variables

**Build:**
- Backend: Standard Cargo.toml with `[dependencies]` and `[dev-dependencies]`
- Frontend: `next.config.ts` (minimal), `tsconfig.json` with `@/*` path alias, `tailwind.config.ts` (auto-generated by shadcn)

## Platform Requirements

**Development:**
- Rust 1.77+ (stable channel)
- Node.js 18+ LTS
- SQLite 3 (bundled with libSQL)
- Turso account (optional for cloud sync, local-only mode available)

**Production:**
- Docker/Linux container runtime
- Turso cloud database (or local SQLite for edge deployments)
- Cloudflare tunnel per client (network routing — not in stack, but deployment requirement)
- One-command shell script install via Docker Compose
