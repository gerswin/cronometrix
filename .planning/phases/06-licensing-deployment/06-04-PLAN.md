<objective>
Build the DO Functions serverless license server: two web-accessible Node.js 22 functions (`activate`, `renew`) that look up license records, validate hardware fingerprint binding, and sign RS256 JWTs. Includes deployment config (`project.yml`), local test harness, and operator README covering key generation, DO secret setup, and `doctl serverless deploy` flow.

Purpose: Server-side authority for LIC-03. Without this, Plan 01's `activate_license` has nothing to call. The DO Functions side controls when a license becomes active and which fingerprint it binds to — anti-cloning defense in depth (paired with Rust's runtime fingerprint check in LIC-05).

Output: Deployable DO Functions package. Operator runs `doctl serverless deploy` and gets a base URL. The DO_FUNCTIONS_ACTIVATE_URL config var on each client server points at `{base_url}/licenses/activate` and `DO_FUNCTIONS_RENEW_URL` at `{base_url}/licenses/renew`.

POST /licenses/activate
- Headers: Content-Type: application/json
- Body: { "license_key": "XXXX-XXXX-XXXX-XXXX", "hardware_fingerprint": "<64-hex-char SHA256>" }

- Response 404: license_key not found in store
- Response 409: license_key already bound to a DIFFERENT hardware_fingerprint
- Response 400: missing or malformed body
- Response 500: server error (e.g. DB unreachable, private key parse fail)

POST /licenses/renew
- Headers: Content-Type: application/json
- Body: { "license_key": "...", "hardware_fingerprint": "..." }
- Response 200: { "token": "<RS256 JWT, refreshed exp>" }
- Response 403: license_key not bound to the supplied fingerprint (anti-cloning)
- Response 404: license_key not found
- Response 400/500: as above

    **2. do-functions/packages/licenses/activate/index.js** — handler:
    ```javascript
    // POST /licenses/activate
    // Body: { license_key, hardware_fingerprint }
    // Response 200: { token }
    // Response 4xx: { error: { code, message } }
    //
    // Persistence: process.env.DATABASE_URL points to Postgres with table:
    //   licenses(license_key TEXT PRIMARY KEY,
    //            hardware_fingerprint TEXT,
    //            activated_at BIGINT,
    //            last_renewed_at BIGINT)
    //
    // For local tests, a process.env.TEST_STORE injection bypasses Postgres.

    const jwt = require('jsonwebtoken');

    const ONE_YEAR_SECS = 365 * 24 * 60 * 60;

    function getStore() {
        // Test mode: in-memory map injected via TEST_STORE
        if (process.env.TEST_STORE) {
            return require('../shared-store');
        }
        // Production mode: lazy-load pg (DO Functions runtime installs from package.json)
        const { Client } = require('pg');
        return {
            async lookup(licenseKey) {
                const client = new Client({ connectionString: process.env.DATABASE_URL });
                await client.connect();
                try {
                    const r = await client.query(
                        'SELECT hardware_fingerprint FROM licenses WHERE license_key = $1',
                        [licenseKey],
                    );
                    if (r.rows.length === 0) return null;
                    return r.rows[0].hardware_fingerprint || null;
                } finally {
                    await client.end();
                }
            },
            async bind(licenseKey, fingerprint, now) {
                const client = new Client({ connectionString: process.env.DATABASE_URL });
                await client.connect();
                try {
                    await client.query(
                        'UPDATE licenses SET hardware_fingerprint = $1, activated_at = COALESCE(activated_at, $2) WHERE license_key = $3',
                        [fingerprint, now, licenseKey],
                    );
                } finally {
                    await client.end();
                }
            },
        };
    }

    function sign(licenseKey, hardwareFingerprint, privateKey) {
        const now = Math.floor(Date.now() / 1000);
        const payload = {
            license_key: licenseKey,
            hardware_fingerprint: hardwareFingerprint,
            product: 'cronometrix',
            iat: now,
            exp: now + ONE_YEAR_SECS,
        };
        return jwt.sign(payload, privateKey, { algorithm: 'RS256' });
    }

    exports.main = async function (args) {
        // DO Functions sometimes parses JSON into top-level args, sometimes nests under `body`.
        // Handle both shapes defensively.
        const body = args && args.body && typeof args.body === 'object' ? args.body : args;
        const license_key = body && body.license_key;
        const hardware_fingerprint = body && body.hardware_fingerprint;

        if (!license_key || !hardware_fingerprint) {
            return {
                statusCode: 400,
                body: { error: { code: 'BAD_REQUEST', message: 'license_key and hardware_fingerprint required' } },
            };
        }

        const privateKey = process.env.LICENSE_PRIVATE_KEY;
        if (!privateKey) {
            return {
                statusCode: 500,
                body: { error: { code: 'CONFIG_ERROR', message: 'license server misconfigured' } },
            };
        }

        try {
            const store = getStore();
            const existingFp = await store.lookup(license_key);
            if (existingFp === null && !(await storeExists(store, license_key))) {
                return {
                    statusCode: 404,
                    body: { error: { code: 'LICENSE_NOT_FOUND', message: 'license key not found' } },
                };
            }

            // Idempotent re-activation on same hardware
            if (existingFp && existingFp !== hardware_fingerprint) {
                return {
                    statusCode: 409,
                    body: { error: { code: 'ALREADY_ACTIVATED', message: 'license already bound to different hardware' } },
                };
            }

            // Bind (no-op if already bound to same fp)
            const now = Math.floor(Date.now() / 1000);
            await store.bind(license_key, hardware_fingerprint, now);

            const token = sign(license_key, hardware_fingerprint, privateKey);
            return { statusCode: 200, body: { token } };
        } catch (e) {
            // Never leak DB error details
            return {
                statusCode: 500,
                body: { error: { code: 'SERVER_ERROR', message: 'license activation failed' } },
            };
        }
    };

    // Distinguish "row exists with null fp" from "row missing entirely"
    async function storeExists(store, licenseKey) {
        // For test store, store.exists is provided.
        if (typeof store.exists === 'function') {
            return await store.exists(licenseKey);
        }
        // For pg store, do an explicit COUNT
        const { Client } = require('pg');
        const client = new Client({ connectionString: process.env.DATABASE_URL });
        await client.connect();
        try {
            const r = await client.query('SELECT 1 FROM licenses WHERE license_key = $1', [licenseKey]);
            return r.rows.length > 0;
        } finally {
            await client.end();
        }
    }

    // POST /licenses/renew
    // Body: { license_key, hardware_fingerprint }
    // Response 200: { token }  — fresh exp = now + 1y
    // Response 403: fingerprint mismatch (anti-cloning at server)
    // Response 404: license not found

    const jwt = require('jsonwebtoken');

    const ONE_YEAR_SECS = 365 * 24 * 60 * 60;

    function getStore() {
        if (process.env.TEST_STORE) {
            return require('../shared-store');
        }
        const { Client } = require('pg');
        return {

                    await client.end();
                }

            async touch(licenseKey, now) {
                const client = new Client({ connectionString: process.env.DATABASE_URL });
                await client.connect();
                try {
                    await client.query(
                        'UPDATE licenses SET last_renewed_at = $1 WHERE license_key = $2',
                        [now, licenseKey],
                    );
                } finally {
                    await client.end();
                }
            },
        };
    }

    exports.main = async function (args) {
        const body = args && args.body && typeof args.body === 'object' ? args.body : args;
        const license_key = body && body.license_key;
        const hardware_fingerprint = body && body.hardware_fingerprint;

        if (!license_key || !hardware_fingerprint) {
            return { statusCode: 400, body: { error: { code: 'BAD_REQUEST', message: 'license_key and hardware_fingerprint required' } } };
        }

        const privateKey = process.env.LICENSE_PRIVATE_KEY;
        if (!privateKey) {
            return { statusCode: 500, body: { error: { code: 'CONFIG_ERROR', message: 'license server misconfigured' } } };
        }

        try {
            const store = getStore();
            const existingFp = await store.lookup(license_key);
            if (existingFp === undefined) {
                return { statusCode: 404, body: { error: { code: 'LICENSE_NOT_FOUND', message: 'license key not found' } } };
            }
            if (existingFp !== hardware_fingerprint) {
                return { statusCode: 403, body: { error: { code: 'HARDWARE_MISMATCH', message: 'license bound to different hardware' } } };

            const now = Math.floor(Date.now() / 1000);
            await store.touch(license_key, now);

            const payload = {
                license_key,
                hardware_fingerprint,
                product: 'cronometrix',
                iat: now,
                exp: now + ONE_YEAR_SECS,

            const token = jwt.sign(payload, privateKey, { algorithm: 'RS256' });
            return { statusCode: 200, body: { token } };
        } catch (e) {
            return { statusCode: 500, body: { error: { code: 'SERVER_ERROR', message: 'license renewal failed' } } };
        }

    # Cronometrix License Server

    DigitalOcean Functions serverless implementation of the license activation
    and renewal endpoints. Consumed by the Rust API's `license::service::activate_license`

    ## One-time setup

    1. **Install doctl**: <https://docs.digitalocean.com/reference/doctl/how-to/install/>

    2. **Generate the production RSA keypair** (once, kept in a vault):

       openssl genrsa -out license_private.pem 2048
       openssl rsa -in license_private.pem -pubout -out license_public.pem
       ```
       - Copy `license_public.pem` to `backend/src/license/pubkey.pem` and rebuild the API.
       - Store `license_private.pem` ONLY as a DO Functions env var. Do NOT commit.

    3. **Provision a license database** (DO Managed Postgres, $15/mo cheapest):
       ```sql
       CREATE TABLE licenses (
         license_key TEXT PRIMARY KEY,
         hardware_fingerprint TEXT,
         activated_at BIGINT,
         last_renewed_at BIGINT
       );
       ```
       Pre-seed each customer license:
       ```sql
       INSERT INTO licenses (license_key, hardware_fingerprint) VALUES ('XXXX-XXXX-XXXX-XXXX', NULL);
       ```

    4. **Set deploy-time env vars**:
       ```bash
       export LICENSE_PRIVATE_KEY="$(cat license_private.pem)"
       export DATABASE_URL="postgres://user:pass@host:5432/licenses?sslmode=require"
       ```

    5. **Deploy**:
       ```bash
       cd do-functions
       doctl serverless deploy . --remote-build
       ```

    6. **Get the URLs**:
       ```bash
       doctl serverless functions get licenses/activate --url
       doctl serverless functions get licenses/renew --url
       ```
       Set these as `DO_FUNCTIONS_ACTIVATE_URL` and `DO_FUNCTIONS_RENEW_URL` in each
       client server's `/opt/cronometrix/.env` after install.

    ## Architecture notes

    - RS256 algorithm is hardcoded in `jwt.sign(...)` and the Rust verifier hardcodes
      `Algorithm::RS256`. Swapping algorithms requires changes in both places (defense
      in depth against alg=HS256 substitution).
    - License records are seeded by the operator BEFORE customer install; activation
      only binds the fingerprint, never creates new keys.
    - Fingerprint mismatch on renewal returns 403, mirroring the Rust binary's

      the renewal path won't issue new tokens.

      in the DO Functions Node.js 22 runtime.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → DO Functions | Public web endpoints invokable by anyone. Mitigations live entirely in handler logic — no auth header, the license_key + fingerprint binding IS the auth. |
| DO Functions runtime → process.env.LICENSE_PRIVATE_KEY | DO platform stores the secret encrypted at rest; runtime exposes via process.env. Trust boundary is DigitalOcean's secret store. |
| DO Functions → Postgres | Connection over TLS (sslmode=require in DATABASE_URL). Postgres credentials stored as DO env var alongside private key. |
| Function → Rust client (response) | Response body contains JWT; the JWT is not a secret per se (it's bearer-issued to the requesting machine), but the private key MUST never appear in any response. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-37 | Spoofing | Replay attack — same body POSTed twice | mitigate | Idempotent by design: same license_key + same fingerprint = same outcome (re-issues a valid JWT for that hardware). License lifecycle is monotonic; operator revokes via DB update. |
| T-06-38 | Tampering | License key brute force / enumeration | mitigate | License keys are 16-alphanumeric-char (XXXX-XXXX-XXXX-XXXX) → 36^16 ≈ 8e24 keyspace. Even at 100 req/s sustained, full enumeration would take ~2.5e15 years. DO Functions also rate-limits free-tier accounts. Production should add fail2ban-style rate limiting via DO Functions middleware (deferred for v1). |
| T-06-39 | Repudiation | License binding decisions have no audit trail | mitigate | `activated_at` + `last_renewed_at` columns in licenses table provide a coarse audit. Full audit (who, when, IP) is deferred per CONTEXT deferred ideas (license analytics out of scope). |
| T-06-40 | Information Disclosure | Private key extraction from logs | mitigate | Handler NEVER logs the key (no console.log near jwt.sign). DO Functions platform redacts `process.env.*` references in logs by default. The catch block returns generic SERVER_ERROR — never the exception's stack. |
| T-06-41 | Information Disclosure | DB credentials in logs | mitigate | DATABASE_URL accessed only inside getStore(); pg client closes the connection in finally. No SQL errors leak to response body — caught and replaced with SERVER_ERROR. |
| T-06-42 | Information Disclosure | Stolen JWT replay on a third-party machine | mitigate | The JWT carries hardware_fingerprint; Rust client (Plan 01) re-computes fp at startup and rejects mismatched JWTs. DO Functions /renew also re-checks fp before issuing a fresh JWT. Two-tier defense. |
| T-06-43 | Denial of Service | Floods exhausting DO Functions free tier | accept | DO Functions has its own platform-level abuse protections (rate limits, cold start budget). Cost is bounded. v1 accepts DOS exposure. |
| T-06-44 | Tampering | Algorithm confusion: client sending alg=none | mitigate | Server side does NOT verify incoming JWTs (it issues them). Rust verifier is the receiver, and Plan 01's verify_license_jwt hardcodes Algorithm::RS256 (test_verify_rejects_hs256_token proves it). |
