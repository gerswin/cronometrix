Use a debian-slim runner (NOT alpine) because libsql links against glibc in the standard build; alpine + musl requires extra cross-compilation work that adds risk. RESEARCH § Standard Stack lists alpine:3.21 — explicitly deviate here and note it in the SUMMARY.

    ```dockerfile
    # syntax=docker/dockerfile:1.7
    # Phase 6: Cronometrix API container (Rust 1.93 + Axum 0.8.8 + libSQL).
    # Multi-stage build. Final image is debian-slim because libsql dynamically
    # links to glibc; using musl/alpine would require extra effort and ships no
    # observable size win after stripping. ~80MB compressed final image.

    FROM rust:1.93 AS builder
    WORKDIR /app

    # Pre-cache deps: copy ONLY manifests, fetch, then copy source.
    COPY backend/Cargo.toml backend/Cargo.lock* ./
    RUN mkdir -p src && echo "fn main() {}" > src/main.rs && \
        cargo build --release --bin cronometrix-api && \
        rm -rf src

    COPY backend/ ./
    # Re-touch main so cargo recompiles with real source
    RUN touch src/main.rs && cargo build --release --bin cronometrix-api

    # Runner: debian-slim with CA certs (needed for Turso TLS + DO Functions HTTPS)
    FROM debian:bookworm-slim AS runner
    RUN apt-get update && \
        apt-get install -y --no-install-recommends ca-certificates curl && \
        rm -rf /var/lib/apt/lists/*

    WORKDIR /opt/cronometrix
    COPY --from=builder /app/target/release/cronometrix-api /usr/local/bin/cronometrix-api

    # Data directory for SQLite + license.jwt; created by installer outside container
    # but ensured here so the binary's default paths work even without a volume mount.
    RUN mkdir -p /opt/cronometrix/data

    EXPOSE 3001
    ENV RUST_LOG=info \
        SERVER_HOST=0.0.0.0 \
        SERVER_PORT=3001 \
        CRONOMETRIX_DB_PATH=/opt/cronometrix/data/cronometrix.db \
        LICENSE_JWT_PATH=/opt/cronometrix/data/license.jwt \
        TZ=America/Caracas

    HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
        CMD curl -fsS http://localhost:3001/api/v1/health || exit 1

    CMD ["/usr/local/bin/cronometrix-api"]
    ```

    ```yaml
    # Cronometrix v1 deployment — Phase 6
    # Services: api (Rust/Axum), web (Next.js), cloudflared (Cloudflare Tunnel)
    # Generated/copied by deploy/install.sh; lives at /opt/cronometrix/docker-compose.yml
    # on each client server. .env file in the same directory provides secrets.

    services:
      api:
        image: ghcr.io/cronometrix/api:${VERSION:-latest}
        restart: unless-stopped
        env_file: .env
        environment:
          - CRONOMETRIX_DB_PATH=/opt/cronometrix/data/cronometrix.db
          - LICENSE_JWT_PATH=/opt/cronometrix/data/license.jwt
          - SERVER_HOST=0.0.0.0
          - SERVER_PORT=3001
        ports:
          # Bind to localhost only — cloudflared (or operator's reverse proxy)
          # is the only entry point from the public internet.
          - "127.0.0.1:3001:3001"
        volumes:
          - ./data:/opt/cronometrix/data
        healthcheck:
          test: ["CMD", "curl", "-fsS", "http://localhost:3001/api/v1/health"]
          interval: 30s
          timeout: 5s
          retries: 3
          start_period: 15s

      web:
        image: ghcr.io/cronometrix/web:${VERSION:-latest}
        restart: unless-stopped
        environment:
          - NEXT_PUBLIC_API_URL=http://api:3001
          - NODE_ENV=production
        ports:
          - "127.0.0.1:3000:3000"
        depends_on:
          api:
            condition: service_started

      cloudflared:
        image: cloudflare/cloudflared:2026.3.0
        restart: unless-stopped
        # `tunnel run` with no tunnel name uses the embedded TUNNEL_TOKEN to
        # connect to the pre-created tunnel in CF Zero Trust (D-13).
        command: tunnel --no-autoupdate run
        environment:
          # `${VAR:?msg}` syntax: docker compose hard-fails if unset, surfacing
          # config errors at boot instead of silently running a broken tunnel
          # (RESEARCH § Pitfall 6).
          - TUNNEL_TOKEN=${CLOUDFLARE_TUNNEL_TOKEN:?CLOUDFLARE_TUNNEL_TOKEN required — re-run installer or edit .env}
        # No `depends_on: web` constraint — DEPL-04: api/web must boot even if
        # cloudflared is unavailable (e.g. operator activated offline).
    ```

    **2. deploy/install.sh** — `set -euo pipefail`, idempotent, 3-prompts-plus-tunnel-token. The installer:
    1. Verifies prerequisites (docker, docker compose plugin, openssl, curl, python3).
    2. Prompts: license key, client slug, admin password, Cloudflare tunnel token.
    3. Creates /opt/cronometrix and /opt/cronometrix/data.
    4. Generates secrets (only if .env doesn't exist — Pitfall 7 idempotency).
    5. Writes .env, copies docker-compose.yml from a release artifact (or generates inline).
    6. Calls DO Functions to activate license; persists JWT to /opt/cronometrix/data/license.jwt.
    7. `docker compose up -d`.
    8. Health-checks api on http://127.0.0.1:3001/api/v1/health.

    Note: For the "fingerprint" used in the activation call, the installer runs `docker run --rm ghcr.io/cronometrix/api:latest fingerprint` BEFORE `compose up` — but that requires a `fingerprint` subcommand in the binary (Plan 01 didn't add one to keep scope tight). Resolution: the installer writes the license_key and CLIENT_SLUG to .env, runs `docker compose up -d api`, then issues `curl -X POST http://127.0.0.1:3001/api/v1/setup/activate` with the license key — the API's own fingerprint code computes the canonical fp. This eliminates RESEARCH Pitfall 3 (installer/Rust fingerprint mismatch) by making the API the single source of truth.

    Final installer:
    ```bash
    #!/usr/bin/env bash
    # Cronometrix one-command installer (DEPL-01).
    # Usage:  curl -sSL https://install.cronometrix.com/install.sh | sudo bash
    # Or:     sudo bash deploy/install.sh
    #
    # Idempotent: re-runs detect existing /opt/cronometrix/.env and skip secret
    # regeneration so user sessions survive (Pitfall 7).

    set -euo pipefail

    # ---------------------------------------------------------------- config
    INSTALL_DIR="${CRONOMETRIX_INSTALL_DIR:-/opt/cronometrix}"
    DATA_DIR="${INSTALL_DIR}/data"
    ENV_FILE="${INSTALL_DIR}/.env"
    COMPOSE_FILE="${INSTALL_DIR}/docker-compose.yml"
    TZ_DEFAULT="${CRONOMETRIX_TZ:-America/Caracas}"
    IMAGE_REGISTRY="${CRONOMETRIX_IMAGE_REGISTRY:-ghcr.io/cronometrix}"
    VERSION="${CRONOMETRIX_VERSION:-latest}"

    # ---------------------------------------------------------------- helpers
    log() { printf '[install] %s\n' "$*" >&2; }
    err() { printf '[install] ERROR: %s\n' "$*" >&2; exit 1; }

    require_cmd() {
        if ! command -v "$1" >/dev/null 2>&1; then
            err "missing required command: $1"
        fi
    }

    require_root() {
        if [ "$(id -u)" -ne 0 ]; then
            err "must run as root (sudo)"
        fi
    }

    # ---------------------------------------------------------------- preflight
    log "checking prerequisites..."
    require_root
    require_cmd docker
    require_cmd openssl
    require_cmd curl
    require_cmd python3
    if ! docker compose version >/dev/null 2>&1; then
        err "docker compose plugin not installed (need 'docker compose', not 'docker-compose')"
    fi

    # ---------------------------------------------------------------- prompts
    log "Cronometrix installer — 4 inputs required"
    if [ -t 0 ]; then
        # Interactive
        read -r -p "License key (XXXX-XXXX-XXXX-XXXX): " LICENSE_KEY
        read -r -p "Client slug (becomes {slug}.cronometrix.com): " CLIENT_SLUG
        read -r -s -p "Admin password (min 8 chars): " ADMIN_PASSWORD
        echo
        read -r -p "Cloudflare tunnel token (from CF Zero Trust dashboard): " CF_TOKEN
    else
        # Non-interactive — read from env vars (CI/CD)
        LICENSE_KEY="${CRONOMETRIX_LICENSE_KEY:?CRONOMETRIX_LICENSE_KEY required in non-interactive mode}"
        CLIENT_SLUG="${CRONOMETRIX_CLIENT_SLUG:?CRONOMETRIX_CLIENT_SLUG required in non-interactive mode}"
        ADMIN_PASSWORD="${CRONOMETRIX_ADMIN_PASSWORD:?CRONOMETRIX_ADMIN_PASSWORD required in non-interactive mode}"
        CF_TOKEN="${CRONOMETRIX_CF_TUNNEL_TOKEN:?CRONOMETRIX_CF_TUNNEL_TOKEN required in non-interactive mode}"
    fi

    # Validate license key shape (XXXX-XXXX-XXXX-XXXX, 19 chars)
    if [ "${#LICENSE_KEY}" -ne 19 ] || ! echo "$LICENSE_KEY" | grep -qE '^[A-Za-z0-9]{4}-[A-Za-z0-9]{4}-[A-Za-z0-9]{4}-[A-Za-z0-9]{4}$'; then
        err "license key must be in XXXX-XXXX-XXXX-XXXX format"
    fi
    if [ "${#ADMIN_PASSWORD}" -lt 8 ]; then
        err "admin password must be at least 8 characters"
    fi
    if ! echo "$CLIENT_SLUG" | grep -qE '^[a-z0-9][a-z0-9-]{1,62}[a-z0-9]$'; then
        err "client slug must be 3-64 chars, lowercase alphanumeric + hyphens, no leading/trailing hyphen"
    fi

    # ---------------------------------------------------------------- directories
    log "creating ${INSTALL_DIR} and ${DATA_DIR}"
    mkdir -p "$INSTALL_DIR" "$DATA_DIR"
    chmod 750 "$INSTALL_DIR" "$DATA_DIR"

    # ---------------------------------------------------------------- secrets (idempotent)
    if [ -f "$ENV_FILE" ]; then
        log "${ENV_FILE} exists — preserving existing JWT_SECRET / DEVICE_CREDS_KEY"
        # shellcheck disable=SC1090
        set -a; . "$ENV_FILE"; set +a
        : "${JWT_SECRET:=}"
        : "${DEVICE_CREDS_KEY:=}"
        if [ -z "$JWT_SECRET" ] || [ -z "$DEVICE_CREDS_KEY" ]; then
            err "existing $ENV_FILE missing JWT_SECRET or DEVICE_CREDS_KEY — back it up and remove to regenerate"
        fi
    else
        log "generating fresh secrets via openssl rand"
        JWT_SECRET="$(openssl rand -hex 32)"
        DEVICE_CREDS_KEY="$(openssl rand -base64 32 | tr -d '\n')"
    fi

    # ---------------------------------------------------------------- write .env
    log "writing ${ENV_FILE}"
    umask 077  # 0600 perms — secrets file
    cat > "$ENV_FILE" <<EOF
    # Cronometrix runtime config — generated by install.sh
    # Do NOT commit this file. chmod 600 enforced.
    JWT_SECRET=${JWT_SECRET}
    DEVICE_CREDS_KEY=${DEVICE_CREDS_KEY}
    CLOUDFLARE_TUNNEL_TOKEN=${CF_TOKEN}
    CLIENT_SLUG=${CLIENT_SLUG}
    LICENSE_JWT_PATH=/opt/cronometrix/data/license.jwt
    TZ=${TZ_DEFAULT}
    CRONOMETRIX_DB_PATH=/opt/cronometrix/data/cronometrix.db
    SERVER_HOST=0.0.0.0
    SERVER_PORT=3001
    VERSION=${VERSION}
    EOF
    chmod 600 "$ENV_FILE"
    umask 022

    # ---------------------------------------------------------------- compose file
    if [ ! -f "$COMPOSE_FILE" ]; then
        log "writing ${COMPOSE_FILE}"
        # The installer ships a copy of docker-compose.yml alongside itself.
        # If invoked from the repo, copy from ./deploy/docker-compose.yml.
        # If invoked via curl|bash, the compose YAML is fetched from the same release.
        SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]:-$0}")" 2>/dev/null && pwd)"
        if [ -f "${SCRIPT_DIR}/docker-compose.yml" ]; then
            cp "${SCRIPT_DIR}/docker-compose.yml" "$COMPOSE_FILE"
        else
            # Fetch from release artifact
            curl -fsSL "https://install.cronometrix.com/docker-compose.yml" -o "$COMPOSE_FILE"
        fi
    fi

    # ---------------------------------------------------------------- pull + start api first
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator → install.sh | Operator runs script as root; trusted to provide accurate inputs. |
| install.sh → /opt/cronometrix/.env | Plaintext secrets on disk; chmod 600 + root-owned + INSTALL_DIR chmod 750 limit lateral exposure. |
| install.sh → DO Functions (via api container) | Activation traffic; HTTPS via reqwest rustls. |
| Internet → cloudflared | Cloudflare Tunnel token authenticates the daemon to the pre-created tunnel; tunnel routes to web:3000. |
| Docker build context → image layers | Anything not in .dockerignore lands in image; secrets must be excluded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-25 | Information Disclosure | Secrets baked into Docker image layers | mitigate | `.dockerignore` excludes `.env`, `*.pem` (with explicit allowlist for `backend/src/license/pubkey.pem` only), `**/data/`, `**/tests/`. License private key NEVER exists in repo (D-01). |
| T-06-26 | Information Disclosure | install.sh leaking secrets in shell history | mitigate | `read -r -s -p` for admin password (no echo); license key + tunnel token are not echoed; `set +o history` is intentionally NOT used because operator may want audit trail of which install.sh path executed. |
| T-06-27 | Information Disclosure | install.sh leaving secrets in /tmp or world-readable .env | mitigate | `umask 077` before writing .env; `chmod 600 $ENV_FILE` after write; INSTALL_DIR mode 750 (root rwx, group rx, others none). No /tmp scratch files. |
| T-06-28 | Tampering | curl|bash MITM | mitigate | Operator served from cronometrix.com domain over HTTPS (Cloudflare CDN). v1 does not implement signature verification of the script (deferred). Operator may instead clone the repo and run `sudo bash deploy/install.sh` for higher assurance. |
| T-06-29 | Elevation of Privilege | install.sh running as root and writing user data | mitigate | Script enforces `id -u == 0` via require_root() — fails fast if not root. INSTALL_DIR is created with explicit chmod; SQLite/JWT files inherit root ownership. |
