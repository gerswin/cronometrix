Create the database module with embedded migrations:

1. **backend/src/db/mod.rs** — Database initialization and migration runner:
   - `pub async fn init_db(config: &Config) -> anyhow::Result<libsql::Database>` function:
     a. Build database with `libsql::Builder::new_remote_replica()` using config.database_path, config.turso_url, config.turso_token
     b. Set `.sync_interval(Duration::from_secs(config.turso_sync_interval_secs))`
     c. Set `.read_your_writes(true)`
     d. `.build().await`
     e. Get connection: `db.connect()?`
     f. Apply PRAGMAs on local connection BEFORE sync: `PRAGMA foreign_keys = ON;`
     g. Call `run_migrations(&conn).await?`
     h. Sync: `db.sync().await?`
     i. Return db
   - Migration runner: `MIGRATIONS` const array of `(&str, &str)` tuples using `include_str!()` to embed SQL files
   - `_migrations` tracking table: `CREATE TABLE IF NOT EXISTS _migrations (name TEXT PRIMARY KEY, applied_at INTEGER NOT NULL)`
   - For each migration: check if already applied (SELECT COUNT), if not: `conn.execute_batch(sql).await?`, insert into `_migrations`
   - Log each applied migration with `tracing::info!`
   - Also provide `pub async fn init_db_local(config: &Config) -> anyhow::Result<libsql::Database>` for local-only mode (when Turso vars are empty): use `Builder::new_local(&config.database_path)` — enables development without Turso credentials

| Boundary | Description |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I (Info Disclosure) | .env file | mitigate | .gitignore excludes .env; .env.example has placeholder values only |
| T-01-02 | T (Tampering) | audit_log table | mitigate | No DELETE or UPDATE triggers on audit_log; SQLite triggers cannot be bypassed by app code |
| T-01-03 | S (Spoofing) | JWT_SECRET | mitigate | Config panics at startup if JWT_SECRET is missing or < 32 chars |
| T-01-04 | D (Denial of Service) | migration runner | accept | Migrations run once at startup; idempotent via _migrations table |
</threat_model>
