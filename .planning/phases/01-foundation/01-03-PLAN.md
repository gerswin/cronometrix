3. **backend/src/employees/service.rs** — Database operations:
   - `pub async fn create(conn, req) -> Result<Employee, AppError>`:
     a. Validate department_id exists: `SELECT id FROM departments WHERE id = ?1 AND status = 'active'` — return NotFound with code "DEPARTMENT_NOT_FOUND" if missing (EMP-04)
     b. Generate UUID v4 for id
     c. `INSERT INTO employees (id, employee_code, name, department_id, status, version, created_at, updated_at) VALUES (?1, ?2, ?3, ?4, 'active', 1, unixepoch(), unixepoch())`
     d. Handle UNIQUE constraint violation on employee_code — return Conflict with code "EMPLOYEE_CODE_EXISTS"
     e. Return created employee

   - `pub async fn list(conn, query) -> Result<PaginatedResponse<Employee>, AppError>`:
     a. Build dynamic WHERE clause: always include `status = 'active'` unless query.status is specified
     b. Filter by name with `LIKE '%' || ?N || '%'` if query.name present
     c. Filter by department_id with `= ?N` if present
     d. Count total: `SELECT COUNT(*) FROM employees WHERE ...`
     e. Fetch page: `SELECT * FROM employees WHERE ... ORDER BY name ASC LIMIT ?N OFFSET ?N`
     f. Clamp limit to 1..100, default 20; offset default 0
     g. Convert all epoch integers to ISO 8601 strings using `common::epoch_to_iso()` per D-13

   - `pub async fn get_by_id(conn, id) -> Result<Employee, AppError>`:
     a. `SELECT * FROM employees WHERE id = ?1`
     b. Return NotFound with code "EMPLOYEE_NOT_FOUND" if no row

   - `pub async fn update(conn, id, req) -> Result<Employee, AppError>`:
     a. Build dynamic SET clause from non-None fields
     b. Always set `updated_at = unixepoch()`, `version = version + 1`
     c. `UPDATE employees SET ... WHERE id = ?1 AND version = ?2` (D-04 optimistic concurrency)
     d. If 0 rows affected, return Conflict with code "VERSION_CONFLICT"
     e. Return updated employee

   - `pub async fn deactivate(conn, id) -> Result<(), AppError>`:
     a. `UPDATE employees SET status = 'inactive', deleted_at = unixepoch(), updated_at = unixepoch(), version = version + 1 WHERE id = ?1 AND status = 'active'`
     b. If 0 rows affected, return NotFound with code "EMPLOYEE_NOT_FOUND"
     c. Per D-03: soft delete uses both status=inactive AND deleted_at timestamp
