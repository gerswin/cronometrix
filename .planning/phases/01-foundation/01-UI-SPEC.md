# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 frontend deliverable is a single screen — the first-boot setup wizard for admin account creation (D-07 from CONTEXT.md). No dashboard, no tables, no navigation. The full UI system is built in Phase 4. This contract establishes the design foundation that Phase 4 must inherit without contradiction.

## Interaction Contract

### Setup Wizard Flow

1. Next.js middleware checks `GET /api/v1/setup/status` on every request.
2. If `{ configured: false }` — redirect all routes to `/setup`.
3. If `{ configured: true }` — redirect `/setup` to `/login`; block re-access.
4. `/setup` renders a single centered card with the admin creation form.
5. On submit: POST `/api/v1/setup/init` with form payload.
6. On success (201): redirect to `/login`.
7. On validation error (400/422): display field-level errors inline below each input.
8. On server error (500): display a top-of-card error banner with the server error message.

### Form Behavior

- All fields required — validated client-side with Zod before submit.
- Submit button disabled while request is in flight (loading state).
- Password field: type=password with toggle visibility button (lucide-react `Eye` / `EyeOff` icon).
- No auto-focus tricks — browser default focus on first field is sufficient.
- Form does not reset on server error — preserve user input.

### Loading Skeleton (Claude's Discretion — D from CONTEXT.md)

- While middleware checks `/api/v1/setup/status` on initial load: show a centered spinner (shadcn `Loader2` icon, `animate-spin`, 24px, accent color).
- No skeleton bones needed — single card, fast check, spinner is sufficient.

### Error States

- Field validation errors: red text (destructive color), 14px/400 weight, displayed below the input with 4px margin.
- Top-of-card server error banner: destructive background at 10% opacity, destructive border-left 4px, full-width inside card, 16px padding, `AlertCircle` icon (lucide-react) + message text inline.
- Already-configured redirect: show "Setup is already complete. Redirecting to login…" centered on page for 1.5 seconds, then redirect — no card needed.

## Layout Contract

```
viewport (full height, flex column, justify-center, align-center)
  └── Card (max-width: 448px, width: 100%, shadow-md)
        ├── CardHeader (padding: lg=24px)
        │     ├── Logo or product name (optional, placeholder)
        │     ├── CardTitle: "Create your admin account" (heading 24px/600)
        │     └── CardDescription: subheading (body 16px/400, muted color)
        └── CardContent (padding: lg=24px, flex column, gap: md=16px)
              ├── FormField: Full name (Label + Input)
              ├── FormField: Username (Label + Input)
              ├── FormField: Password (Label + Input[type=password] + toggle)
              ├── FormField: Confirm password (Label + Input[type=password] + toggle)
              ├── [Error banner — conditional]
              └── Button (full width, "Create account", accent bg)
```

Card top-of-page centering: `min-h-screen flex items-center justify-center` on the page wrapper. Horizontal padding: `px-4` on mobile so card does not touch viewport edges.

## Accessibility Contract

- All form inputs have associated `<label>` elements (not placeholder-as-label).
- Error messages are associated with inputs via `aria-describedby`.
- Submit button has `aria-disabled` when loading, not HTML `disabled` (preserves focus ring for keyboard users).
- Color contrast: shadcn/ui defaults pass WCAG AA at 4.5:1 minimum for text on backgrounds.
- Password toggle button has `aria-label="Show password"` / `"Hide password"`.
