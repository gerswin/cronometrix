<objective>
Build the complete backend for facial enrollment and multi-device sync. This plan delivers
the database schema, ISAPI client extensions, enrollment module with multipart receive +
server-side downscale, JoinSet-based concurrent push, two background workers (purge on
deactivation, backfill on new device registration), audit triggers, and full integration
test coverage. Implements ENRL-01..05 backend surface; the frontend (07-02) consumes the
five endpoints exposed here.

Purpose: Close Phase 7 backend so admins can enroll faces from any of three sources (device
kiosk, webcam JPEG, JPG upload) and have the system fan out to every Hikvision device
concurrently with per-device status visibility, while keeping inactive employees off devices
and new devices automatically synced with active employees.

Output: Migrations 016/017, `enrollments/` module (mod, models, service, handlers, pusher,
image_pipeline, isapi_face), `workers/` module (purge, backfill), extended `isapi/client.rs`,
extended `state.rs`/`main.rs`, deactivation/registration hooks in employees/devices handlers,
Wave 0 test scaffolds + populated tests, Bruno collection for the five new endpoints.
</objective>
