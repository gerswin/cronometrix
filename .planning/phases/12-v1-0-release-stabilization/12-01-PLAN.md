# Rebaseline and Reproducible Test Harness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: use `superpowers:executing-plans` task-by-task and `superpowers:test-driven-development` for behavioral changes.

**Goal:** establish a truthful planning baseline and a deterministic Node/Rust/Playwright harness from which every remaining product failure is reproducible.

**Architecture:** normalize toolchains and dependency locks, remove process-global test races, give every Playwright test a fresh authenticated session, and make local and CI build the same binaries, database, frontend mode, and run identifier. Mechanical format/lint cleanup lands separately from semantic harness fixes.

**Tech Stack:** Rust nightly pinned by `rust-toolchain.toml`, Cargo/Clippy/rustfmt, Node 24.15.0, npm 11.12.1, Next.js, Vitest, Playwright, GitHub Actions.

### Step 1: Write deterministic reset tests first

Rewrite `test_reset_gating.rs` so it never calls `set_var`/`remove_var`. Assert disabled+absent route is 404, enabled+registered is 200, registered+disabled is 404, and enabled reset performs cleanup.


### Step 2: Inject E2E mode once at startup

Add to `AppState`:

```rust
/// Startup-captured test-mode capability. Never read process environment at use-site.
pub e2e_enabled: bool,
```

Read `CRONOMETRIX_E2E` once in `main.rs`, use the boolean for license bypass, state construction, and route registration. Replace the handler's environment read with:

```rust
if !state.e2e_enabled {
    return Err(StatusCode::NOT_FOUND);
}
```

Default it to false in `backend/tests/common/mod.rs`; individual tests set their newly constructed state before router cloning.

### Step 3: Isolate license child processes

Create a `TempDir` in each license safety test and set `.current_dir(tmp_dir.path())` on every `Command`, while retaining `env_clear`, explicit required environment, unique database/license paths, and exit-code-2 assertions. This prevents `dotenvy` from loading `backend/.env`.

### Step 3: Decouple release mode from generic CI

Use `CRONOMETRIX_E2E_RELEASE=true` for release backend binaries plus `next start`; otherwise use debug plus `next dev`. `make e2e-build` must build the selected backend mode and Next.js before `next start`. `make e2e` exports one run ID and calls the same path CI uses.

### Step 4: Replace reusable cookies with fresh per-test auth

Delete static `.auth/*.json` generation. `frontend/e2e/fixtures/auth.ts` exports:

```ts
export type E2ERole = 'admin' | 'supervisor' | 'viewer'

export const ROLE_CREDENTIALS: Record<
  E2ERole,
  { username: string; password: string }
>

export async function newRoleContext(
  browser: Browser,
  role: E2ERole,
): Promise<BrowserContext>
```

`newRoleContext` creates an empty context, logs in through `context.request`, asserts success, and returns the context carrying its fresh refresh cookie. Never persist storage state.

Also export an extended Playwright `test` whose default role is admin, whose `context` fixture calls `newRoleContext`, and whose `request` fixture uses `context.request`. Authenticated specs import this fixture. `login.spec.ts` remains on base Playwright. One-off viewer/supervisor tests create and close fresh role contexts.

This is mandatory because the backend stores one refresh-token hash per user and rotates it. Browser single-flight in 12-02 cannot repair token reuse across test contexts.
