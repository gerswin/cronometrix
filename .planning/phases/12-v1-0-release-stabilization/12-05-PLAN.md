# Full Technical Release Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: use `superpowers:executing-plans` task-by-task and `superpowers:verification-before-completion` before recording a candidate.

**Goal:** converge formatting, lint, executable tests, queue-only enforcement, coverage, build, E2E, and container smoke into one local command and five required GitHub checks on one immutable release-candidate SHA.

**Architecture:** eliminate ignored scaffold tests, strengthen the same-origin smoke from 12-04, expose composable Make targets, and use a local `release-gate` orchestrator that refuses a dirty or moving HEAD. CI keeps the established coverage/E2E names, adds Container Smoke, and ends with an aggregate Release Gate. A pinned candidate branch and private release bundle are created from the passing SHA; its summary lives after that SHA and is never confused with the tested commit.

**Tech Stack:** Cargo/rustfmt/Clippy/cargo-llvm-cov/nextest, npm/TypeScript/Vitest/Next.js, Playwright, Docker Compose, Bash, GitHub Actions.

## Global Constraints

- Depends on 12-01 plus the scoped checkpoints from 12-02, 12-03, and 12-04
  integrated into one clean branch. This plan must consume and close every
  item in their coverage/deferred-risk ledgers before an unqualified PASS.
- Do not change product behavior merely to make a gate green; route fixes back to the owning plan and rerun the entire gate.
- Do not add ignored scaffolds, fake assertions, blanket lint allows, coverage exclusions, threshold reductions, or CI `continue-on-error` for required checks.
- Keep exactly two justified ignored backend tests: implemented Turso live sync and the real 1,000-employee performance benchmark.
- All five required CI checks must run against the same SHA.
- A failure invalidates prior candidate evidence; fix in a separate commit and rerun from the beginning.
- Run tests before every commit.

### Step 1: Keep these exact required check names

```text
Backend Coverage
Frontend Coverage
E2E Tests
Container Smoke
Release Gate
```

Backend Coverage runs rustfmt, strict Clippy, test-stubs, queue-only, then coverage. Frontend Coverage runs exact Node/npm, `npm ci`, typecheck, build, then coverage. E2E builds Next.js before `next start`. Container Smoke invokes the local Make target and never authenticates to GHCR.

Release Gate is a lightweight `needs` aggregator over the other four. It runs with `if: always()` and fails unless every needed result is `success`. Serialize the results to JSON and assert with jq; do not rerun the suites in the aggregator.

All jobs keep `permissions: contents: read`. Test/log/report uploads use `if: always()` and 14-day retention. Preserve coverage scope parity between Make and CI.

### Step 2: Build the final private candidate bundle

Push a unique one-use release-build ref at this exact SHA. The push triggers both `Release` and `CI` even while those workflow revisions exist only on the candidate commit. Poll both by workflow name, event, branch, SHA, and trigger time; never select `gh run list --limit 1`:

```bash
set -euo pipefail
source /tmp/cronometrix-12-05-candidate.env
test "$(git status --porcelain)" = ""
test -n "$CANDIDATE_SHA"
test "$(git rev-parse HEAD^{commit})" = "$CANDIDATE_SHA"
RELEASE_BUILD_BRANCH="codex/release-build-${CANDIDATE_SHA}"
test -z "$(git ls-remote --heads origin "refs/heads/${RELEASE_BUILD_BRANCH}")"
TRIGGERED_AT="$(date -u +%Y-%m-%dT%H:%M:%SZ)"
git push origin "${CANDIDATE_SHA}:refs/heads/${RELEASE_BUILD_BRANCH}"
test "$(git ls-remote origin "refs/heads/${RELEASE_BUILD_BRANCH}" | awk '{print $1}')" = "$CANDIDATE_SHA"
RELEASE_RUN_ID=""
CI_RUN_ID=""
for attempt in $(seq 1 30); do
  if [ -z "$RELEASE_RUN_ID" ]; then
    RELEASE_RUN_ID="$(gh run list --branch "$RELEASE_BUILD_BRANCH" --event push --limit 50 \
      --json databaseId,headSha,createdAt,workflowName \
      --jq "[.[] | select(.workflowName == \"Release\" and .headSha == \"$CANDIDATE_SHA\" and .createdAt >= \"$TRIGGERED_AT\")] | .[0].databaseId // empty")"
  fi
  if [ -z "$CI_RUN_ID" ]; then
    CI_RUN_ID="$(gh run list --branch "$RELEASE_BUILD_BRANCH" --event push --limit 50 \
      --json databaseId,headSha,createdAt,workflowName \
      --jq "[.[] | select(.workflowName == \"CI\" and .headSha == \"$CANDIDATE_SHA\" and .createdAt >= \"$TRIGGERED_AT\")] | .[0].databaseId // empty")"
  fi
  test -n "$RELEASE_RUN_ID" && test -n "$CI_RUN_ID" && break
  sleep 5
done
test -n "$RELEASE_RUN_ID"
test -n "$CI_RUN_ID"
gh run watch "$RELEASE_RUN_ID" --exit-status
gh run watch "$CI_RUN_ID" --exit-status
gh run view "$CI_RUN_ID" --json headSha,event,url,jobs \
  > /tmp/cronometrix-12-05-ci-run.json
jq -e --arg sha "$CANDIDATE_SHA" '
  .headSha == $sha and
  .event == "push" and
  (.jobs | length == 5) and
  (.jobs | map(.name) | sort) ==
    (["Backend Coverage", "Frontend Coverage", "E2E Tests", "Container Smoke", "Release Gate"] | sort) and
  (.jobs | all(.conclusion == "success"))
' /tmp/cronometrix-12-05-ci-run.json
RELEASE_RUN_URL="$(gh run view "$RELEASE_RUN_ID" --json url --jq .url)"
CI_RUN_URL="$(jq -r .url /tmp/cronometrix-12-05-ci-run.json)"
test -n "$RELEASE_RUN_URL"
test -n "$CI_RUN_URL"
printf 'RELEASE_RUN_ID=%q\nRELEASE_RUN_URL=%q\nCI_RUN_ID=%q\nCI_RUN_URL=%q\n' \
  "$RELEASE_RUN_ID" "$RELEASE_RUN_URL" "$CI_RUN_ID" "$CI_RUN_URL" \
  >> /tmp/cronometrix-12-05-candidate.env
RELEASE_TMP="$(mktemp -d)"
gh run download "$RELEASE_RUN_ID" --dir "$RELEASE_TMP"
test "$(find "$RELEASE_TMP" -name '*.tar.gz' | wc -l)" = 1
test "$(find "$RELEASE_TMP" -name '*.tar.gz.sha256' | wc -l)" = 1
find "$RELEASE_TMP" -name '*.tar.gz.sha256' -execdir sha256sum -c "{}" \;
RELEASE_EXTRACTED="$(mktemp -d)"
RELEASE_TAR="$(find "$RELEASE_TMP" -name '*.tar.gz' -print -quit)"
tar -tzf "$RELEASE_TAR" | sort | diff -u <(printf '%s\n' SHA256SUMS docker-compose.yml install.sh nginx.conf release-manifest.env) -
tar -xzf "$RELEASE_TAR" -C "$RELEASE_EXTRACTED"
(cd "$RELEASE_EXTRACTED" && sha256sum -c SHA256SUMS)
rg -n "^SOURCE_SHA=${CANDIDATE_SHA}$" "$RELEASE_EXTRACTED/release-manifest.env"
rm -rf "$RELEASE_EXTRACTED"
rm -rf "$RELEASE_TMP"
```

Expected: the private release workflow, all five Linux CI jobs, and both checksum layers pass on the exact same candidate SHA; the manifest source SHA is exact. Capture both run URLs, five job URLs/conclusions, bundle hash, and four digests for the summary.
