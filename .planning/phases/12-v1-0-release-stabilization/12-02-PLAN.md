# Functional Contracts and Authenticated UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: use `superpowers:executing-plans` task-by-task and `superpowers:test-driven-development` for every behavior change.

**Goal:** repair authentication, device, enrollment, timesheet, and live-dashboard contracts across Rust, TypeScript, unit/integration tests, and Playwright.

**Architecture:** the backend DTO is authoritative; typed frontend clients own paths and serialization. Session bootstrap is explicit and refresh is single-flight. Enrollment capture sources converge on one validated photo state and in-progress work is server-backed. Daily rows are keyed by employee/date. SSE follows the live access token and carries enriched event data.

**Tech Stack:** Axum/libSQL, Next.js/React/TypeScript, Axios, TanStack Query/Table, Vitest/Testing Library, Playwright.

### Step 1: Write the red contract

Add tests proving:

1. two simultaneous 401s call `/auth/refresh` once;
2. both requests retry once with the same new token;
3. failed shared refresh clears token, shows one toast, and redirects once;
4. `_retry=true` never loops;
5. provider starts `initializing`, refreshes from the httpOnly cookie, then becomes authenticated or anonymous;
6. an existing valid memory token skips bootstrap refresh;
7. the gate hides RBAC content while initializing;
8. anonymous redirects once with pathname and query preserved;

### Step 2: Implement the single-flight API

Use this public contract:

```ts
export type AuthStatus = 'initializing' | 'authenticated' | 'anonymous'

export interface AuthContextValue {
  status: AuthStatus
  role: 'admin' | 'supervisor' | 'viewer' | null
  sub: string | null
  claims: JWTClaims | null
}

export function refreshAccessToken(): Promise<string>
```

In `api.ts`, keep exactly one module-level `refreshPromise: Promise<string> | null`. `refreshAccessToken` uses a bare Axios client without the response interceptor, posts to `${API_BASE}/api/v1/auth/refresh` with credentials, stores/returns the access token, and resets the promise in `finally`.

The response interceptor marks `_retry`, waits on the shared promise, rewrites `Authorization`, and retries once. A separate deduplicated session-expired handler performs clear/toast/redirect only for request-time failures. Bootstrap failure is silent and transitions to anonymous.

### Step 3: Implement provider and gate

Subscribe to token changes before bootstrapping. Decode an existing token immediately; otherwise call refresh once. `SessionGate` renders `data-testid="session-initializing"` during bootstrap and redirects from `useEffect` only after anonymous is established. Wrap the dashboard layout with it.
