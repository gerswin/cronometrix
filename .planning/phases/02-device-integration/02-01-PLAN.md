<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → `/api/v1/devices/*` | Admin-only mutation surface; JWT validated in middleware; DoS risk on command dispatch |
| backend/device row → `devices/crypto.rs` decrypt | Encrypted blob + AES-GCM key cross the boundary in-memory only; plaintext MUST NOT be logged or returned |
| Rust backend → Hikvision device (outbound ISAPI) | Untrusted device may return malicious response bodies; digest auth over TLS (self-signed opt-in per allow_insecure_tls) |
| Admin JWT → door_open command | Elevation surface: a stolen admin token lets the attacker open physical doors; mitigation = short access token TTL (Phase 1 D-06) + every dispatch written to command_audit_log |
