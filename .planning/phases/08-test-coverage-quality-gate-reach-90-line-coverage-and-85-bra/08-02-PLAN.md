<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test → tempdir | Each test owns a per-test `TempDir`; no cross-test filesystem leakage |
| test → AppState | Tests pass `Paths::for_test(tmp.path())` explicitly; no env-var mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Information Disclosure | Test-side env mutation | mitigate | Remove all `*RootGuard` env-var manipulation. Each test owns a fresh `TempDir` whose contents drop at scope end. Eliminates the previously-existing class where one test's env-var value could leak into another's filesystem operation under `cargo nextest`'s parallel runner. |
| T-08-06 | Tampering | TempDir lifetime | mitigate | `test_state_with_tmpdir` returns `(AppState, TempDir)` as a tuple so the caller is forced to acknowledge tempdir ownership. Doc comment explicitly warns about Pitfall 1 (premature drop). |
| T-08-07 | Repudiation | Audit-log integrity in tests | accept | Audit-log triggers and on-disk evidence behaviors are unchanged. The refactor swaps the path source, not the audit semantics. Existing tests that assert on audit_log rows continue to assert; the only change is the path under which evidence files land. |
</threat_model>
