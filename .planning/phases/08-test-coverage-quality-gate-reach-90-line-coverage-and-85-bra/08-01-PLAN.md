<objective>
Promote filesystem roots (`leaves_root`, `events_root`, `enrollments_root`,
`captures_tmp_root`, plus the inline `DATA_DIR`/`overrides` read in
daily_records) from cwd-dependent, process-global env-var lookups into a
`Paths` substruct held by `AppState`. Per D-17/D-18/D-19/D-21: production
startup keeps reading the same env vars with the same defaults at startup
once; handlers and services read the paths from `&state.paths.<field>` at
use-site instead of calling per-call helpers.

Purpose: Eliminate the cwd-dependent + env-race anti-pattern that breaks
`leave_tests` under `cargo-llvm-cov` and any test runner that changes cwd
or runs in parallel. This is Wave 1 because every later wave (test-helper
migration, coverage measurement, CI gate) depends on the refactor landing
cleanly first.

Output: `backend/src/state/paths.rs` (new), `backend/src/state.rs` (new
field), `backend/src/main.rs` (wiring), and 5 source files updated to
read paths from `state.paths.*`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → AppState | Process env vars (LEAVES_ROOT, EVENTS_ROOT, ENROLLMENTS_DIR, CAPTURES_TMP, DATA_DIR) are read once at startup and become immutable Paths fields |
| AppState → handler | State<AppState> extractor delivers paths to handlers; handlers no longer read env at use-site |
| handler → filesystem | Existing path-traversal guards (canonicalize, prefix-check) at consumer sites remain unchanged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `Paths::from_env` | mitigate | Read env once at startup; defaults are repo-controlled relative paths matching the deleted helpers verbatim (D-21 backwards compat). No path normalization or canonicalization changed — preserves existing consumer-side guards. |
| T-08-02 | Information Disclosure | `Paths` Debug impl | accept | Paths contain no secrets (filesystem-root strings); `#[derive(Debug)]` is safe. Rationale: paths are not credentials, and the existing `Config` Debug redaction (which IS needed for secrets) is unchanged. |
| T-08-03 | Tampering (path traversal) | leaves/events/daily_records handlers | mitigate | Existing canonicalize + prefix-check logic in `get_leave_evidence` and equivalents is preserved verbatim. The refactor only swaps the root-source; it does not weaken traversal guards. Verification: grep for `canonicalize` / `starts_with` in handlers post-refactor returns the same matches as pre-refactor. |
| T-08-04 | Repudiation | env-var-driven path injection in tests | mitigate | Removing `LeavesRootGuard`/`EventsRootGuard` eliminates the env-var-mutation race that previously could let one test's env value leak into another's filesystem operations under parallel execution. Tests now own their tempdir explicitly. |
</threat_model>
