<objective>
Open a deliberately-red pull request that exercises all 3 CI gate failure paths (Backend Coverage post-processor, Frontend Coverage Vitest threshold, E2E Tests Playwright assertion) and capture the failing-job logs as evidence that the gates work as designed. Close the PR without merging.

Purpose: A green run alone (Plan 11-01) only proves the happy path. To trust the gates as a hard-fail guarantee for v1.0 we also need proof the negative path triggers — i.e. when code coverage drops or a spec breaks, the runner refuses the merge. This is Phase 8 Plan 05's deferred step B and Phase 9 Plan 12's deferred step 2 in one PR.

Output: 1 closed PR with the 3-failure diff + 3 raw failure logs + verdict README in `evidence/03-ci-red/`. No source code in `main` is modified — the PR is closed, not merged (D-15).

This plan is NOT autonomous: an operator must (1) create the regression branch with the exact 3 breaking changes, (2) push + open PR, (3) wait for CI, (4) capture failure logs, (5) close PR. Executor validates that all 3 jobs FAIL for the right reason (not a flake or unrelated infra failure — D-17).
</objective>
