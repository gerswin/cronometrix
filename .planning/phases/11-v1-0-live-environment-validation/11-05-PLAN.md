<objective>
On a fresh Ubuntu 22.04 VM with Docker, run the v1.0 `curl | bash` installer end-to-end with real Cloudflare tunnel TOKEN + DO Functions license URL, and capture committed evidence that DEPL-01, DEPL-02, DEPL-04, LIC-01..LIC-04 all work in a real deploy environment (not just unit tests).

Purpose: Phase 6 verified the installer with synthetic tests + manual one-shot dev runs, but the formal first-fresh-VM smoke was deferred for milestone v1.0 sign-off. This plan executes that smoke once on a real VM with real third-party endpoints (Cloudflare + DO Functions) and saves enough raw output that future re-runs (or post-mortems if a deploy regresses) have a known-good baseline.

Output: 7 evidence files in `evidence/05-installer-smoke/` capturing every phase of the install — VM spec, installer stdout, docker-compose state, API health, license activation, cloudflared tunnel registration — plus a verdict README. Total size budget: under 5 MB (text-only artifacts; no HTML reports).

This plan is NOT autonomous: an operator must (1) provision a VM, (2) run the installer, (3) capture each phase's output, (4) tear down the VM after evidence is committed. Executor's role: generate the per-phase capture commands, validate that each captured file contains the expected greppable markers, and write the verdict README.

This plan is an independent track from the CI-gate validation chain (11-02 / 11-04). It depends only on 11-01 to scaffold the shared evidence directory; once 11-01 is committed, 11-05 can run in parallel with 11-02 and 11-03.
</objective>
