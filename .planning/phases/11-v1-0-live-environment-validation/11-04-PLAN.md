<objective>
Toggle GitHub branch protection on `main` to require all 3 CI status checks (`Backend Coverage`, `Frontend Coverage`, `E2E Tests`) before merge, and capture both visual (screenshot) and API (gh api JSON) evidence proving the rule is active. This closes Phase 8 Plan 05's deferred step 3 + Phase 9 Plan 12's deferred step 3 in one operator session.

Purpose: Without branch protection enforcement, the CI gates are advisory — a maintainer can still merge red. Branch protection is the policy layer that converts the "hard-fail in CI" property into "cannot merge into main". This plan is the final lock that makes the v1.0 quality gates a true safeguard.

Output: 1 PNG screenshot of the GitHub Settings → Branches UI + 1 JSON file from `gh api` showing `required_status_checks` includes the 3 contexts, packaged with a verdict README in `evidence/04-branch-protection/`.

This plan is NOT autonomous: an operator must (1) navigate to the GitHub Settings → Branches page, (2) toggle the protection rule, (3) screenshot it, (4) run `gh api` for machine-readable verification. Executor validates that the JSON contains all 3 expected check names and that the screenshot file is present and non-zero-byte.

This plan REQUIRES Plans 11-01 + 11-02 to have completed first because the GH UI dropdown for "required status checks" only lists checks that have actually run on a recent commit. Without the live green run (11-01) and the live red run (11-02), the 3 checks may not appear in the picker.
</objective>
